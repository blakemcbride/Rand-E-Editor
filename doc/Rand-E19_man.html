NAME

e - Rand full screen text editor  

SYNOPSIS

e [ options ] [ file ] [ altfile ] [ other file ] ....
ned  

DESCRIPTION

e invokes the Rand editor. The Rand editor is a full screen text editor with file editing in windows on the screen. Editing is done by typing directly into the window, and functions include "cut" and "paste" operations, simple formatting, and embedding UNIX command output in the file.

The parameter file denotes the file to be edited and altfile is an optional alternate file. With no file names given the editor will resume the latest editing session in the directory, if any, otherwise it will prompt for a file name.

E is a complete rewrite of the Rand editor NED The current version (E19) takes advantage of terminals that have the ability to do local insert/delete line. A reference manual The Rand Editor, E, April 1984 is available on the PRIAM VAX using the notesfile scheme and the printdoc command.  

CHANGES

To see the last changes in version E19 : look at the output of "help new" command.

E19 (initial) changes include:

- word processing mode (WP mode)
- a delete word function
- options to control the behavior at the right margin;
- an alternate syntax for the "goto" command;
- display of tab settings on the top border
- left and right margin (width) settings on the bottom border
- an option for invoking E with a previous state file
- a new help facility;
- improved support for the "track" command;
- new commands for case conversion and reversal.
E18 changes include:

- support for regular expression searching and replacment
- implementation of the +WORD and -WORD functions
- a new set command to individually tailor various parameters
- support for symbolic links
- E now runs under UNIX System III and System V
- commands that specify filenames (e.g., " CMD: e file" expand a leading "~" to the appropriate home directory;
 

OPTIONS

The currently available command line options are:

-help

This causes a display of the currently available options and their values along with the version number of the editor.
Use this option to list the updated list of options.
-bullets or -nobullets

Options -bullets and -nobullets forces border bullets on or off regardless of line speed or terminal type. Line speeds of 1200 baud and slower and some terminal types set border bullets off in the absence of these options.
-inplace

Option -inplace preserves file links except for singly-linked files. From within the editor, link preservation can be turned on and off, even for singly-linked files, by the update command.
-terminal=termtype

Option -terminal=termtype identifies the terminal type explicitly. In the absence of the -terminal option the terminal type is retrieved from the TERM environment variable.
-keyboard=termtype or -dkeyboard=termtype

Option -keyboard=termtype sets the keyboard type to "termtype". It overrides any "-terminal=" argument and also any EKBD environment variable.
-dtermcap

Option -dtermcap forces E to use the termcap entry for the terminal identified by the TERM variable or -terminal option, disregarding any applicable compiled-in code.
-notracks

Option -notracks will allow you to edit a file without using or disturbing the work files (keys, state, and change files) from your previous session.
-package=directory_name

Option -package=directory_name allows you to overwrite the build in default directory in which the Rand editor is looking to find service programs (like just, fill) and messages and information files (like Crashdoc, helpkey). In the absence of the -package option the Rand package directory is retrieved from the RAND_PACKG environment variable.
-replay=filename

Option -replay=filename allows you to replay with a specified keys file, where "filename" represents that keys file. Use -silent when the replay display is not desired. The message "***replay completed" or "***replay aborted" signals the end of the replay. The <INT> key can be used to stop the replay short of completion. Files .ec1* and .ek1* must not be present if the replay is to work.
-norecover

Option -norecover will open the edit session as defined by the .es1 state file, ignoring any recovery processing of a prior crashed or aborted session.
-regexp

Option -regexp enables regular expression pattern matching for searching and replacing. This mode is also enabled by the "re[gexp]" command. In regular expression mode, "RE" is displayed at the bottom of the screen. The command "-re[gexp]" returns to non-regular expression mode. The regular expressions recognized are essentially the same as that for ed.
-state=filename

Option -state=filename causes the initial state of the editor to be read from the indicated "state" file. The state file from a previous E session (with alternate files, windows, cursor positioning, etc.) may be saved by exiting, and renaming the state file (".es1").
-dump_state_file

Option -dump_state_file display in a readable format the content of the "state" file. This state file can be explicitely define by the "-state=filename" option or is the state file which will be used when the editor is called without file name parameter.
-[no]stick

Option -stick sets the behavior at the right margin to stop in response to typing or a <RT ARROW>. -nostick is the default, and will automatically shift the window right in response to text or <RT ARROW> key.
-nox11

Option -nox11 disables calls to X11 library during startup to get the keyboard mapping.
-debug=[debug output file name]:[debug level]

Option -debug starts a debugging session, level = 0 : debugging suspended, level != 0 : debugging actif. Use the command 'set debuglevel x' to suspend, resume the debugging. When the debugging is suspended, the debugging output file is flused, and closed. When debugging is resumed, entering in a level != 0, the debugging output file is reopened for append. On starting the debugging session, the sizeof the major Rand data type (like Char, Flag, ...) is dumped on the debugging file.
-version

Option -version display editor version informations. It is equivalent to the -help option.
 

TERMINAL TYPES

The terminal type is defined by the option -terminal if given, otherwise by the TERM environment variable.

The mapping of the logical keys onto physical terminal keys is also defined by the terminal type, unless the EKBD environment variable is set to standard or the option -keyboard=standard is given. This option overrides the EKBD variable, if any.

The instructions to drive the terminal may be coded in the editor or may be stored in the termcap(5) database. The parameter -dtermcap forces the use of termcap(5).

The following terminal types are recognised:

Pericom - pa
Pericom with 132 columns - pa132
Pericom graphics terminal - vt100
DEC VT100 terminal - vt100
DEC VT220 terminal - vt200
DEC VT220 dense mode - vt200-w
SUN Workstation - sun
Newbury 7001 - nb7
TANDBERG 2200 - tdv

where the right hand side gives the terminalid.

EDITOR FUNCTIONS

 

 

Terminology

Keys denoted by upper case text are logical keys. A logical key has a unique meaning to the editor and is mapped on a physical key of a terminal as given in the tables above.

The following generic items are used:

filename
UNIX file name, shell metacharacters are not expanded
text

string of characters, may include blanks
number
decimal number in the range 1 - 32767
cursormovement
key sequences that result in a movement of the cursor
linearea
one or more entire lines. Specified either as a decimal number immediately followed by l meaning lines, or as decimal number followed by p meaning paragraphs. A paragraph is terminated by an empty line or end of file.
area

a rectangular part of the file, defined by MARK followed by cursor movements.

[ and ] (square brackets) surround optional items. All other character strings are to be taken literally.

Key functions

It is better to look at the output of "help keyf" for the updated description of the key functions.

ALT - swap current file and alternate file.

BACKSPACE - move cursor one position left.
ENTER BACKSPACE - move cursor to left edge of window.

BOX - draw a box around the boundary of a

marked area

MARK cursormovement BOX

BTAB - move cursor left to previous tabulator stop.

CAPS - convert lower case characters to upper case

in the marked area. Only characters (a-z) are affected. An area must be defined

CCASE - change case (lower -> upper and vice-versa)

in the marked area. An area must be defined

CENTER - center lines beginning at the cursor

If an area is marked, the marked area of integer lines is centered. Otherwise one line is centered.

CHWIN - make next window the current window.
ENTER number CHWIN - make window number the current window.

CONTROL X CONTROL C - the next character entered will be entered as

a control character.

DELCH - delete the character at the cursor.
ENTER DELCH - delete all characters from the cursor to the

end of the line. The most recently deleted characters are kept in the erase buffer if not in insert state, otherwise in the close buffer.

DELETE - if not in insert state:

write a blank space to the left of the cursor and move cursor one position left,
in insert state:
delete the character to the left of the cursor and move the cursor one position left.
ENTER DELETE - if not in insert state:
replace the part of the line to the left of the cursor by blanks, put the replaced text in the erase buffer,
in insert state:
delete the text to the left of the cursor and put the deleted text in the close buffer.

DELLN - delete the line with the cursor.
ENTER number DELLN - delete number lines starting at the cursor.
MARK cursormovement DELLN

- delete the marked area.
The information deleted by the most recent DELLN is kept in the close buffer.
ENTER DELLN - insert the contents of the close buffer before
the line with the cursor if it is entire line(s), otherwise at the cursor position.

DOWN - move cursor one position down.
ENTER DOWN - move cursor to the bottom of window

DWORD - delete the current word or the next word

if the cursor is positioned on a space

ENTER - set command mode.
ENTER ENTER ALT - recall latest command.

ERASE - blank out (replace by spaces) lines or

marked area By default, the cursor line is blanked out.

EXIT - exit from the editor FILL - fill lines beginning at the cursor

Fill moves words so that each line is filled with words up to the right margin. Fill operates on paragraphs or on areas if one has been defined.

HELP - invoke the Rand editor help facility

HOME - move cursor to upper left corner of window.
ENTER HOME - move cursor to lower left corner of window.

INSCH - set and reset insert state.

In insert state the typed in characters are inserted in front of the character at the cursor.

INSLN - insert a line in front of the line with

the cursor.
ENTER number INSLN - insert number lines in front of the line with
the cursor.

MARK cursormovement INSLN
- insert a rectangle of blanks, of the size
of the area, with the upper left corner at the cursor.

INT - interrupt the current command.

JOIN - join the next line to the current line

JUSTIFY - justify text beginning at the cursor

If an area is specifed, justify operates on this area. Otherwise one paragraph is justified. The line width is that specified by the most recent center fill or justify command - default 75

LEFT - move cursor one position left.
ENTER LEFT - move cursor to the left edge of the window.

LINEFEED - move cursor one position down.
ENTER LINEFEED - move cursor to the bottom of the window.

LIN- - move window 5 lines up.
ENTER LIN- - move window bottom to the cursor.
ENTER number LIN- - move window number lines up.

LIN+ - move window 5 lines down.
ENTER LIN+ - move window top to the cursor.
ENTER number LIN+ - move window number lines down.

MARK cursormovement - define an area.

The cursor positions before and after MARK are opposite corners of the rectangular area. Vertical cursor movement define an area of entire lines.
ENTER MARK - cancel marking.

QUOTE - enter a control character into the file

Pressing quote followed by a <char> causes control <char> to be entered into the file

PAGE- - move window one window up.
ENTER PAGE- - move window to the beginning of the file.
ENTER number PAGE- - move window number windows up.

PAGE+ - move window one window down.
ENTER PAGE+ - move window to the end of the file.
ENTER number PAGE+ - move window number windows down.

PICK - put the line at the cursor into

the pick buffer.
ENTER number PICK - put number lines, beginning at the line with
the cursor, into the pick buffer.

MARK cursormovement PICK
- put the area into the pick buffer.
ENTER PICK - insert the contents of the pick buffer before
the line with the cursor if it is entire line(s), otherwise at the cursor position.

REDRAW - redraw the current screen

REPL - replace the string just found by the

string given with the most recent replace command.

RETURN - move cursor to the beginning of the next

line in the window.

RIGHT - move cursor one position to the right.
ENTER RIGHT - if in text move cursor to first blank after

the text, otherwise to right edge of window.

SCH- - search backwards for the most recently

given text.
ENTER SCH- - search backwards for the word at the cursor.
Word cannot contain blanks.
ENTER text SCH- - search backwards for text.
If a range is in effect (set by the range command) the searches are limited to the range.

SCH+ - search forwards for the most recently

given text.
ENTER SCH+ - search forwards for the word at the cursor.
Word cannot contain blanks.
ENTER text SCH+ - search forwards for text.
If a range is in effect (set by the range command) the searches are limited to the range.

SPLIT - split current line at the cursor position

into two lines

TAB - move cursor to the next tabulator stop right

WORD+ - move cursor to the next word right

WORD- - move cursor to the previous word left

WIN-> - move window 16 positions right.
ENTER WIN-> - move left edge of window to the cursor.
ENTER number WIN-> - move window number positions to the right.

WIN<- - move window 16 positions left.
ENTER WIN<- - move right edge of window to the cursor.
ENTER number WIN<- - move window number positions to the left.

The left edge of the window cannot move further left than tby the beginning of the lines.

UP - move cursor one position up.
ENTER UP - move cursor to the top edge of the window.  

EDITOR COMMANDS

 

It is better to look at the output of "help cmd" command for the updated description of Editor Commands.


Commands are entered after the ENTER key is pressed and are executed when RETURN is pressed after the command has been typed in. In the following only the part between ENTER and RETURN is given. The command and its shortest possible abbreviation are given. All abbreviations down to the shortest are valid.

box - draw a box around the boundary of a marked area
bo Simple horizontal or vertical lines can be drawn by

marking an area with a single character height or width.

bye - same command as exit.
b

caps <marked area>

- change lower case characters to upper case
Caps only changes the "target text" - defined with the mark command Only characters a-z are affected

ccase <marked area>

- convert case upper -> lower and vice-versa
The marked area is first defined by the mark command.

center [ width=number ] [ number ]
ce - center lines beginning at the

cursor such that there is the same amount of blanks on either side of the text. With no number parameter one line is centered, otherwise number lines are treated. With no number parameter and a marked area of integer lines the area is centered. The lines are centered individually.
A line length of width is assumed, where width is the value specified with the most recent center, fill, or justify command (default is 75). Width may be abbreviated w.
The most recently centered lines are put in the adjust buffer in their original form.

close [ number ] [ linearea ]
clo - delete lines or area.

Without parameters the cursor line is deleted. With number parameter number lines beginning at the cursor are deleted. With parameter linearea the specified number of paragraphs or lines are deleted. With a marked area and no parameters the area is deleted.
The most recently deleted information is kept in the close buffer.

-close - insert the contents of the close buffer.
-cl It is inserted before the cursor line for entire lines

otherwise at the cursor position.

delete - delete the current file.

The file is deleted from the system on exit from the editor. The delete can be nullified by exit with a parameter.

dword - delete word
-dword If the cursor is "on" a word, that word is deleted

otherwise the next word is deleted.
The deleted word can be restored by -dword.

edit [ filename ]
e - edit the file filename.

Filename becomes the current file and the current file becomes the alternate file. The ALT key switches between them.
\dit - remove the current file from the list of edited files if it was not modified, nor referenced.

erase [ number ] [ linearea ]
er - blank out lines or area.

Without parameters the cursor line is blanked out (replaced by blanks). With number parameter number lines beginning at the cursor are blanked out. With parameter linearea the specified number of paragraphs or lines are blanked out. With a marked area and no parameters the area is blanked out.
The most recently blanked out information is kept in the erase buffer.

-erase - insert the contents of the erase buffer.
-e It is inserted before the cursor line

for entire lines, otherwise at the cursor position.

exit | ex | bye | b [ nosave | quit | abort | dump ]

- Exit the editor.
The different exit options affect whether or not the editor updates the edited files or not, and whether the ".e" state files are changed. The following table give the complete exit story.

            Saves  Update  Remove     Remove

            Files  State   Keys file  Change file

exit         X      X       X          X

exit nosave  -      X       X          X

exit quit    -      -       X          X

exit abort   -      -       -          X

exit dump    -      -       -          -

blot       adjust | close | erase | pick | run | box

-blot      adjust | close | erase | pick | run | box

cover      adjust | close | erase | pick | run | box

insert     adjust | close | erase | pick | run | box

overlay    adjust | close | erase | pick | run | box

underlay   adjust | close | erase | pick | run | box

All of these commands take the named buffer (one of adjust, close, erase, pick, run, or box) and add or merge it into the current file.

Insert takes the named buffer, moves the existing text to make room for it, and then inserts it at the current text location. It is identical to the -pick command, except that now the other named buffers can also be inserted.

Cover places the named buffer over top of the text without moving anything.

Overlay is like cover, except that only the printing characters in the buffer clobber the original text.

Underlay is the reverse of overlay, i.e. the buffer covers only the non-printing characters of the original text.

Blot is like overlay except that positions in the file corresponding to printing characters in the buffer are erased.

-blot is like blot except that positions in the file corresponding to blanks in the buffer are erased.

fill [ width=number ] [ linearea ]
fi - fill lines, beginning at the cursor,

such that there is the same amount of blanks on either side of the text.
With parameter linearea the specified lines or paragraphs are filled. With no linearea parameter one paragraph is filled. With no linearea parameter and a marked area of integer lines the area is filled.
A line length of width is assumed, where width is the value specified with the most recent center, fill, or justify command (default 75). Width may be abbreviated w.
The most recently filled lines are put in the adjust buffer in their original form.

feed [<range>] command-string

- start up a UNIX shell
A Unix shell is started with the command-string, and the text in <range> is supplied to the shell as the standard input. Any results of the command are inserted before the first line in the range.

goto [ b | e | number | rbeginning | rend ]
g - position cursor on line specified.

Parameter number specifies a sequential line number, parameter b means begin of file, parameter e means end of file, and no parameter means begin of file.
Rbeginning and rend specify the beginning and end of a range defined by the range command. They are valid even if the range has been turned off. Their shortest abbreviations are rb and re,respectively.

insert [ adjust | close | erase | pick | run | box ]
i - insert contents of specified buffer at cursor position.


The buffers are filled by the following commands:
adjust - center, fill, justify.
close - DELLN, close, and when in insert
state also:
erase, ENTER DELCH, ENTER DELETE.
Lost text from join.
erase - erase, and when in insert state also:
ENTER DELCH and ENTER DELETE.

pick - PICK, pick.
run - run.
box - box is a buffer that always
contains a box around the previously marked area.

join - join the next line to the current.

The first non blank character of the next line is placed after the last non blank character on the current line, or at the cursor if it is farther to the right.
This mode of operation may be overridden by marking the positions on the two lines that should be joined. Any lost text is put in the close buffer.

justify [ width=number ] [ linearea ]
ju - justify lines, beginning at the cursor such that the last

word on the lines ends at the right margin.
With parameter linearea the specified lines or paragraphs are justified. With no linearea parameter one paragraph is justified. With no linearea parameter and a marked area of integer lines the area is justified.
A line length of width is assumed, where width is the value specified with the most recent center, fill, or justify command (default 75). Width may be abbreviated w.
The most recently justified lines are put in the adjust buffer in their original form.

name filename

- rename the current file
The file being edited is renamed. The renaming takes effect when the editor is exited normally. It may be nullified by exiting with a parameter.

open [ number ] [ linearea ]
op - insert lines or area.

Without parameters an empty line is inserted before the cursor line. With number parameter, number empty lines are inserted before the cursor line. With parameter linearea the specified number of paragraphs or lines are inserted before the cursor line. With a marked area and no parameters a blank filled area of that shape is inserted at the cursor.

pick [ number ] [ linearea ]
pi - pick lines or area.

Without parameters the cursor line is put into the pick buffer. With number parameter, number lines beginning at the cursor are picked. With parameter linearea the specified number of paragraphs or lines are picked. With a marked area and no parameters the area is picked. The most recently picked information is kept in the pick buffer.

-pick - insert the contents of the pick buffer.
-p It is inserted before the line with the cursor for

entire lines, otherwise at the cursor position.

range [ number ] [ linearea ]
ra - define a range for search and replace.

Searches and replaces are limited to the defined range. Number and linearea specify the range from the cursor. One of them must be given unless an area has been marked, then the range is the area.

-range - turn off previously specified range. The range no longer
-ra limits searches and replaces, but it remains defined.

?range - show status of range.
? The status (current or dormant) and extent of the

range is displayed.

redraw - redraw the screen.
red The current screen is redrawn, free of possible messages

not coming from the editor.

replace [ number ] [ linearea ] [ show | interactive ] /texta/textb/
rep - replace texta by textb

from cursor and towards the end of the file.
If a number or linearea are given, or if an area has been marked, the replacements are restricted to the area, otherwise they are made from the cursor position to the end of the file. If a range is in effect (set by the range command) the replacements are limited to the range.
The show parameter will show the replacements on the screen as they take place.
The interactive parameter will cause the command to stop when a testa has been found. The replacement by textb is then done when the REPL key is pressed, and the search for texta is continued when the SCH+ key is pressed. A replacement may be skipped by pressing SCH+ instead of REPL.
Show and interactive may be abbreviated down to s, and i respectively.
Any character that is not a letter or a digit may be used for delimiting texta and texta.

-replace [ number ] [ linearea ] [ show | interactive ] /texta/textb/
-re - replace texta by textb

from cursor and towards the beginning of the file.
If a number or linearea are given, or if an area has been marked, the replacements are restricted to the area, otherwise they are made from the cursor position to the beginning of the file. If a range is in effect (set by the range command) the replacements are limited to the range.
The show parameter will show the replacements on the screen as they take place.
The interactive parameter will cause the command to stop when a testa has been found. The replacement by textb is then done when the REPL key is pressed, and the search for texta is continued when the SCH- key is pressed. A replacement may be skipped by pressing SCH- instead of REPL.
Show and interactive may be abbreviated down to s, and i respectively.
Any character that is not a letter or a digit may be used for delimiting texta and texta.

regexp - enter regular expression mode for searching & replacing;
-regexp exits regular expression mode.

While in regular expression mode, "RE" is displayed at the bottom of the screen.

run text - execute the text as a UNIX command.
r If there is a marked area of entire lines

it is given to the command as standard input and is replaced by the standard output from the command. If there is no marked area the standard output is inserted before the line with the cursor. The run command is similar to the feed command, except that the marked text is deleted and replaced by the results of running the command-string.

save filename
sa - save a copy of the current file.

The current file including its modifications is copied to filename. Filename must be different from the name of the current file.

update | -update [ inplace | -inplace ]

- editor exit control command
The update command tells the editor what to do on exit with the current file. The update command can be used to specify whether or not to break links, the -update command causes any changes to the current file to be ignored.

window [ filename ]
w - create a window

A new window is created as specified by the cursor position The cursor must be at the top or left edge of the current window. The new window is cut out of the current at the cursor. When filename is given the file will be in the window, otherwise the current file will be displayed.

-window - remove the current window.
-w note that the first window cannot be removed.

wp - enter "word processing" mode
-wp - exit "word processing" mode

A WP is displayed at the bottom of the screen in word processing mode. WP mode enables power typing or "word wrap", where any text that is entered past the right margin (see "set rmar" above) is automatically placed on the following line at the left margin (see "set lmar" above). WP mode also moves the cursor to the left margin in response to a <RETURN>. If WP mode is entered while a marked area is in effect, the boundaries of the marked area become the left and right margin settings.

 

FILES

 

\,filename - comma file.

The file being edited is left unchanged until a normal exit is made from the editor. The file is then copied onto a file with the same name preceded by a , (comma) and thereafter the modifications are applied. If no modifications are made or at abnormal exits (exit with a paramater) the comma file is not produced.
For a file to be changed, it and its directory must both have write permission.

\.ec1 - change file.
All changed text from the file being edited is collected in the change file. The changed text may be accessed as alternate files via the pseudo file names #o and #p, and text may be picked from them.
The #o (old lines) file contains all original versions of changed lines.
The #p (pick) file contains all picked text. Text picked from the #p file is not appended to it.
\.ek1 .ek1b - keystroke files.
The files contain all changes made to the files being edited. The format is such that the changes can be applied when a normal exit is made from the editor.
If an exit is made with parameter dump, or if the editor crashes, the keystroke files can be used for recovering the lost editing session. Recovery is done by invoking the editor without a filename, it will then present recovery options to the user. The file .ek1b is used in the recovery process.
\.es1 - state file.
The state file contains information about the last file edited in the directory, including file name, the place in the file where the cursor were when the editor was terminated, the width value (center, fill, justify), the state (insert or normal), the tabulators and the last search string.
When the editor is called without a file name the state file is used to recreate the situation that existed at the previous termination.
